  <name>Task 1: Create document Server Actions (upload, list, delete, chunks)</name>
  <files>
    src/app/actions/documents.ts

  <action>
    1. Install react-dropzone: `npm install --legacy-peer-deps react-dropzone` (requires --legacy-peer-deps for React 19 peer dep conflict)

    2. Create `src/app/actions/documents.ts` with "use server" directive:

    **uploadDocument(formData: FormData):**
    - Verify session via `getSession()` -- return error if not authenticated
    - Extract file from formData, validate: must have .txt or .md extension, must be non-empty, max 5MB. Reject unsupported formats immediately with `{ success: false, error: "Only .txt and .md files are supported" }`
    - Read file content via `file.text()`
    - Extract title from first `# Heading` in content (fallback: filename without extension). Allow title override via formData.get("title") if provided.
    - Insert document record into Supabase `documents` table (title, content) using `createServiceRoleClient()`
    - Chunk the content using `chunkMarkdown({ title, content })` from existing chunker
    - Generate embeddings using `generateEmbeddings(chunkTexts)` from existing module
    - Insert chunk records into `document_chunks` table with all metadata (document_id, document_title, section_heading, content, chunk_position, total_chunks, embedding)
    - Use `revalidatePath("/admin")` to refresh the document list
    - Return `{ success: true, documentId, chunkCount }` on success
    - Return `{ success: false, error: string }` on failure
    - Wrap in try/catch -- if embedding fails after document insert, delete the document to avoid orphans

    **listDocuments():**
    - Verify session via `getSession()` -- return empty array if not authenticated
    - Query Supabase `documents` table: select id, title, content, created_at
    - Also get chunk count per document: query `document_chunks` with count grouped by document_id, OR use a subquery/join
    - Simplest approach: query documents, then for each document query `document_chunks` count. For <50 docs this is fine.
    - Alternative: use raw SQL via `.rpc()` if a function exists, OR just fetch chunk counts separately.
    - Return array of `{ id, title, createdAt, chunkCount }` sorted by created_at desc
    - Use `createServiceRoleClient()` (service role needed for server-side data access)

    **deleteDocument(documentId: string):**
    - Verify session via `getSession()` -- return error if not authenticated
    - Delete from `documents` table where id = documentId (cascade delete handles chunks via FK constraint in schema)
    - Use `revalidatePath("/admin")` to refresh the list
    - Return `{ success: true }` or `{ success: false, error: string }`

    **getDocumentChunks(documentId: string):**
    - Verify session via `getSession()` -- return empty array if not authenticated
    - Query `document_chunks` where document_id = documentId
    - Select: id, content, section_heading, chunk_position, total_chunks
    - Order by chunk_position ascending
    - Return typed array

    IMPORTANT: Every function must verify session independently (Data Access Layer pattern). Do NOT rely on middleware alone.

    Types to define in the same file (or export separately):
    ```typescript
    interface DocumentListItem {
      id: string
      title: string
      createdAt: string
      chunkCount: number
    }

    interface ChunkListItem {
      id: string
      content: string
      sectionHeading: string
      chunkPosition: number
      totalChunks: number
    }
    ```
  </action>
