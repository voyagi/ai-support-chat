  <action>
    1. Install iron-session: `npm install iron-session`

    2. Create `src/lib/auth/session.ts`:
       - Export `SessionData` interface with `isAuthenticated: boolean` and `createdAt: number`
       - Export `sessionOptions` object: password from `process.env.SESSION_SECRET!`, cookieName `"admin_session"`, ttl 86400 (24h), cookieOptions with httpOnly true, secure based on NODE_ENV, sameSite "lax", path "/"
       - Export async `getSession()` function: `const cookieStore = await cookies()` then `return getIronSession<SessionData>(cookieStore, sessionOptions)`
       - Import `getIronSession` from "iron-session", `cookies` from "next/headers"

    3. Create `src/app/actions/auth.ts`:
       - Add "use server" directive at top
       - Export `login(password: string)` function: compare against `process.env.ADMIN_PASSWORD`, if wrong return `{ success: false, error: "Incorrect password" }`, if correct call `getSession()`, set `isAuthenticated = true` and `createdAt = Date.now()`, call `session.save()`, return `{ success: true }`
       - Export `logout()` function: call `getSession()`, call `session.destroy()`, redirect to "/admin/login"
       - Import `redirect` from "next/navigation", `getSession` from "@/lib/auth/session"

    4. Create `src/middleware.ts` (in project root src/ directory):
       - Optimistic check only (NOT the sole auth gate -- Data Access Layer pattern per CVE-2025-29927)
       - Check if `request.nextUrl.pathname.startsWith("/admin")` AND path is NOT "/admin/login"
       - If no "admin_session" cookie exists, redirect to "/admin/login"
       - Export `config` with matcher `["/admin/:path*"]`
       - Import `NextResponse` and `NextRequest` from "next/server"

    5. Update `.env.example`: Add `ADMIN_PASSWORD=your-admin-password` and `SESSION_SECRET=your-32-char-secret-minimum-length-here` under a new `# Admin` section

    6. Update `next.config.ts`: Add `serverActions: { bodySizeLimit: "5mb" }` to enable larger file uploads in next plan. Do NOT nest under `experimental` -- in Next.js 15.x `serverActions` is a top-level config property.

    IMPORTANT: Do NOT use middleware as sole authentication. Every Server Component and Server Action that accesses admin data MUST call `getSession()` and verify `isAuthenticated`. Middleware is only for fast redirect UX.
  </action>

    - `src/middleware.ts` exists and matches /admin/* routes
    - `.env.example` contains ADMIN_PASSWORD and SESSION_SECRET
  </verify>
