**Create `src/app/widget/layout.tsx`:**

A minimal layout for the widget iframe. This layout is needed because the widget page runs
inside an iframe and needs its own ThemeProvider instance (the root layout's ThemeProvider
doesn't need duplication, but we need to ensure proper theme handling).

```
import { ThemeProvider } from "next-themes";
import type { ReactNode } from "react";

export const metadata = {
  title: "Chat Widget",
  robots: "noindex, nofollow",  // Widget pages should not be indexed
};

export default function WidgetLayout({ children }: { children: ReactNode }) {
  return (
    <ThemeProvider
      attribute="data-theme"
      defaultTheme="light"
      enableSystem={false}
      storageKey="widget-theme"
    >
      {children}
    </ThemeProvider>
  );
}
```

Note: The root layout.tsx already provides `<html>`, `<body>`, and a ThemeProvider. The widget
layout nests inside it. Set `defaultTheme="light"` (widget starts light, parent sends theme
via postMessage). Use a different `storageKey` to avoid conflicts with main app theme.

Wait — the root layout wraps ALL pages including `/widget`. The root layout already has
ThemeProvider. So the widget layout should NOT add a second ThemeProvider. Instead, the widget
page itself will use `useTheme()` from next-themes (provided by root layout) and override
the theme based on postMessage.

Revised approach for `src/app/widget/layout.tsx`:
- Minimal layout that adds `robots: noindex` metadata.
- Adds `overflow-hidden` to prevent double scrollbars in iframe.
- Does NOT duplicate ThemeProvider (root layout provides it).

```tsx
import type { Metadata } from "next";
import type { ReactNode } from "react";

export const metadata: Metadata = {
  title: "Chat Widget",
  robots: "noindex, nofollow",
};

export default function WidgetLayout({ children }: { children: ReactNode }) {
  return <div className="h-screen overflow-hidden">{children}</div>;
}
```

**Create `src/app/widget/page.tsx`:**

A client component that:

1. Renders `<ChatWindow widget={true} />` in a full-height container.
2. On mount, sends `{ type: "WIDGET_READY" }` to parent via `window.parent.postMessage()`.
3. Uses `ResizeObserver` on `document.body` to send `{ type: "RESIZE", data: { height } }`
   whenever content height changes.
4. Listens for `{ type: "THEME_UPDATE", theme: "light" | "dark" }` messages from parent and
   calls `setTheme()` from next-themes to apply the theme.
5. Validates message origin: In production, only accept messages from configured origin. Use
   `process.env.NEXT_PUBLIC_APP_URL` as the allowed origin. In development, accept all origins
   (log a warning).

```tsx
"use client";

import { ChatWindow } from "@/components/chat/ChatWindow";
import { useTheme } from "next-themes";
import { useEffect } from "react";

export default function WidgetPage() {
  const { setTheme } = useTheme();

  useEffect(() => {
    const appUrl = process.env.NEXT_PUBLIC_APP_URL;
    const isProduction = process.env.NODE_ENV === "production";

    // Validate message origin
    const isAllowedOrigin = (origin: string) => {
      if (!isProduction) return true;
      if (!appUrl) return false;
      return origin === appUrl;
    };

    // Listen for messages from parent window
    const handleMessage = (event: MessageEvent) => {
      if (!isAllowedOrigin(event.origin)) return;

      const { type, theme } = event.data ?? {};

      if (type === "THEME_UPDATE" && (theme === "light" || theme === "dark")) {
        setTheme(theme);
      }
    };

    window.addEventListener("message", handleMessage);

    // Notify parent that widget is ready
    window.parent.postMessage({ type: "WIDGET_READY" }, "*");

    // Track content height changes and report to parent
    const observer = new ResizeObserver(() => {
      const height = document.body.scrollHeight;
      window.parent.postMessage(
        { type: "RESIZE", data: { height } },
        "*"
      );
    });

    observer.observe(document.body);

    return () => {
      window.removeEventListener("message", handleMessage);
      observer.disconnect();
    };
  }, [setTheme]);

  return (
    <div className="h-full">
      <ChatWindow widget />
    </div>
  );
}
```

Note on `postMessage` targetOrigin: Using `"*"` for messages FROM iframe TO parent is
acceptable because:
- The iframe doesn't know the parent's origin at build time (it could be any host site).
- The WIDGET_READY and RESIZE messages contain no sensitive data.
- Origin validation is critical in the OTHER direction: parent → iframe messages are validated.
