<objective>
Create the widget loader script that host sites include via a script tag. The loader creates
a floating bubble button and an iframe container, handles open/close toggling, mobile
responsive layout, and listens for postMessage events from the widget iframe.

Purpose: This is the external-facing entry point for the embeddable widget. It must be a
standalone vanilla JavaScript file (no React, no Next.js) that works on any website.

Output: `src/widget-loader/index.ts` source, `rollup.config.mjs` build config, built
`public/widget.js` bundle, and `build:widget` npm script.

<task type="auto">
  <name>Task 1: Create widget loader IIFE source</name>
  <files>src/widget-loader/index.ts</files>
  <action>
Create `src/widget-loader/index.ts` as a self-executing function (IIFE pattern). This is
pure vanilla TypeScript — NO React, NO imports from the Next.js app. Rollup will bundle it
into a standalone script.

**Configuration extraction:**
The script reads configuration from `data-*` attributes on its own `<script>` tag:
- `data-widget-url`: Base URL of the app (e.g., `https://yourapp.com`). Falls back to
  inferring from `script.src` (strip `/widget.js` from the URL).
- `data-theme`: Initial theme (`"light"` or `"dark"`). Defaults to `"light"`.
- `data-position`: Bubble position (`"bottom-right"` only for now). Reserved for future.

**Bubble button:**
- Fixed position: bottom `20px`, right `20px`.
- Size: `60px` x `60px`, border-radius `50%` (circle).
- Background: `#2563eb` (blue-600 equivalent). White chat icon (use SVG inline, not emoji).
- z-index: `999999`.
- Box shadow: `0 4px 12px rgba(0,0,0,0.15)`.
- Cursor: pointer.
- Transition: `transform 0.2s ease` for hover scale effect.
- On hover: `transform: scale(1.1)`.
- When widget is open, show X icon instead of chat icon (toggle visual state).

**SVG icons (inline, no external deps):**
- Chat icon: `<svg viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2"><path d="M21 15a2 2 0 0 1-2 2H7l-4 4V5a2 2 0 0 1 2-2h14a2 2 0 0 1 2 2z"/></svg>`
- Close icon: `<svg viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2"><path d="M18 6L6 18M6 6l12 12"/></svg>`

**Iframe container:**
- Fixed position, initially hidden (`display: none`).
- Desktop layout (>= 768px): bottom `90px`, right `20px`, width `400px`, height `600px`,
  border-radius `12px`, max-width `calc(100vw - 40px)`, max-height `calc(100vh - 110px)`.
- Mobile layout (< 768px): fullscreen — top/left/right/bottom `0`, width/height `100%`,
  border-radius `0`. Add `padding-top: env(safe-area-inset-top)` and
  `padding-bottom: env(safe-area-inset-bottom)` for notched devices.
- z-index: `999998`.
- Box shadow (desktop only): `0 8px 32px rgba(0,0,0,0.2)`.
- Overflow: `hidden`.
- Background: `#ffffff` (loading state before iframe renders).

**Iframe element:**
- Inside the container. Width/height `100%`, border `none`.
- `src`: `${widgetUrl}/widget` (points to the Next.js widget page).
- `allow`: `clipboard-write` (for potential copy features).
- `title`: `"Chat Widget"` (accessibility).

**Toggle behavior:**
- Click bubble → if hidden, show container + set `display: block`. If visible, hide.
- Track open state in a local boolean variable.
- When opening on mobile, optionally hide bubble (fullscreen covers it). Show bubble again on close.

**Responsive layout logic:**
- On `window.resize` and `orientationchange`, call `updateLayout()`.
- `updateLayout()` checks `window.innerWidth < 768`:
  - If mobile: apply fullscreen styles to container.
  - If desktop: apply windowed styles to container.
- Also call `updateLayout()` on initial mount and on toggle open.

**postMessage bridge (parent-side):**
- Listen for `message` events on `window`.
- When `type === "WIDGET_READY"`: Log to console "Widget loaded". Optionally send initial
  theme: `iframe.contentWindow.postMessage({ type: "THEME_UPDATE", theme: currentTheme }, widgetUrl)`.
- When `type === "RESIZE"` and `data.height`: Only relevant if we wanted dynamic height
  (skip for now — container has fixed dimensions). Store for potential future use.
- Origin validation: Check `event.origin` starts with `widgetUrl` before processing messages.
  In production this prevents other iframes from spoofing messages.

**DOM mounting:**
- Append bubble and container to `document.body`.
- Do this at end of IIFE execution (script runs after being appended to DOM).

**Important conventions:**
- All DOM elements use `id` prefixed with `ai-chat-widget-` to avoid conflicts.
- All styles applied via `Object.assign(el.style, {...})` — no CSS classes that could conflict.
- Wrap entire IIFE in `(function() { "use strict"; ... })();` pattern.

<task type="auto">
  <name>Task 2: Configure Rollup build and npm script</name>
  <files>rollup.config.mjs, package.json</files>
  <action>
**Install Rollup dev dependencies:**

```bash
npm install --save-dev rollup @rollup/plugin-typescript @rollup/plugin-terser
```

Use `@rollup/plugin-typescript` (not `@rollup/plugin-node-resolve` or `@rollup/plugin-commonjs`)
because the loader is a single self-contained file with zero imports. TypeScript plugin
compiles TS to JS. Terser minifies for production.

Do NOT install `rollup-plugin-postcss` or `tailwindcss-scoped-preflight` — the widget loader
uses inline styles only, no Tailwind classes. Tailwind is only used inside the iframe (which
is served by Next.js and already has Tailwind).

**Create `rollup.config.mjs`:**

```javascript
import terser from "@rollup/plugin-terser";
import typescript from "@rollup/plugin-typescript";

export default {
  input: "src/widget-loader/index.ts",
  output: {
    file: "public/widget.js",
    format: "iife",
    name: "AIChat",
    sourcemap: false,
  },
  plugins: [
    typescript({
      tsconfig: false,
      compilerOptions: {
        target: "ES2020",
        module: "ESNext",
        moduleResolution: "bundler",
        strict: true,
        lib: ["ES2020", "DOM"],
      },
    }),
    terser({
      mangle: { toplevel: true },
      compress: { drop_console: false, passes: 2 },
    }),
  ],
};
```

Notes:
- `tsconfig: false` tells the plugin not to use the project's tsconfig.json (which has
  Next.js-specific settings). We provide standalone compiler options.
- `drop_console: false` — keep console.log for "Widget loaded" message. Can change to
  `true` later for production.
- Output format `iife` with name `AIChat` wraps everything in a self-executing function.
- No sourcemap for production widget bundles (keeps bundle small).

**Add npm scripts to `package.json`:**

Add these scripts:
- `"build:widget": "rollup -c rollup.config.mjs"`
- Update `"build"` to include widget build: `"build": "rollup -c rollup.config.mjs && next build"`

This ensures `npm run build` produces both the widget.js bundle and the Next.js app.
Rollup runs first (faster, ~1-2s) so the widget.js file is in `public/` before Next.js
copies static files.

**Add widget-loader to biome.json includes:**

Add `"src/widget-loader/**"` to the `files.includes` array in `biome.json` so the loader
code gets linted and formatted by Biome.

**Build the widget:**

Run `npm run build:widget` to verify Rollup produces `public/widget.js`. Check the output
file exists and is under 10KB (should be ~3-5KB minified for pure DOM manipulation code).

**Add `public/widget.js` to `.gitignore`:**

Since it's a build artifact, add `public/widget.js` to `.gitignore` (same as `.next/`).
Actually, for Vercel deployment, the file needs to be in `public/` at build time. The Vercel
build process runs `npm run build` which will generate it. So adding to `.gitignore` is
correct — it gets rebuilt on every deploy.

Wait — if `public/widget.js` is gitignored, it won't be in the repo. Vercel runs `npm run
build` which now includes `rollup -c` first. So Vercel will build it. This is correct.

But for local development, developers need to run `npm run build:widget` once. Add a note
comment in the rollup config.
