**Update `next.config.ts` to allow iframe embedding for the widget page:**

By default, Next.js may set `X-Frame-Options: DENY` or `SAMEORIGIN`, which blocks the
widget page from loading in iframes on other sites. Add custom headers:

```typescript
const nextConfig: NextConfig = {
  experimental: {
    serverActions: {
      bodySizeLimit: "5mb",
    },
  },
  async headers() {
    return [
      {
        // Allow /widget to be embedded in iframes on any site
        source: "/widget",
        headers: [
          {
            key: "X-Frame-Options",
            value: "ALLOWALL",
          },
          {
            key: "Content-Security-Policy",
            value: "frame-ancestors *",
          },
        ],
      },
      {
        // Allow widget.js to be loaded from any site
        source: "/widget.js",
        headers: [
          {
            key: "Access-Control-Allow-Origin",
            value: "*",
          },
        ],
      },
    ];
  },
};
```

Notes:
- `frame-ancestors *` in CSP allows any site to embed the `/widget` page in an iframe.
  This is intentional — the widget is designed to be embedded anywhere.
- `Access-Control-Allow-Origin: *` on `widget.js` allows any site to load the script.
- Only the `/widget` route gets these permissive headers — the rest of the app keeps
  default restrictive headers.
- `X-Frame-Options: ALLOWALL` is deprecated in favor of CSP `frame-ancestors`, but
  including both ensures compatibility with older browsers.
