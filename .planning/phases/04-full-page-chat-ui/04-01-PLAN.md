**ChatWindow.tsx** — The main chat container. This is the most complex component. `'use client'` directive.

**useChat integration:**
- Import `useChat` from `@ai-sdk/react`
- Configure: `api: '/api/chat'`, `body: { conversationId }` (from state)
- Custom `fetch` function that extracts `X-Conversation-Id` and `X-Sources` headers before returning response:
  ```
  fetch: async (url, options) => {
    const response = await fetch(url, options);
    const convId = response.headers.get('X-Conversation-Id');
    const sourcesHeader = response.headers.get('X-Sources');
    if (convId && !conversationId) setConversationId(convId);
    if (sourcesHeader) { /* parse and store per-message sources */ }
    return response;
  }
  ```
- Destructure: `messages`, `sendMessage`, `status`, `error`, `reload` (reload replaces regenerate in AI SDK v6 — verify in docs, fall back to `reload` if `regenerate` not available)

**Source tracking per message:**
- Maintain a `Map<string, Source[]>` or `Record<string, Source[]>` keyed by message ID to store sources for each assistant response
- When X-Sources header arrives during streaming, associate sources with the latest assistant message ID once it appears in messages array
- Pass sources to MessageBubble for assistant messages

**Layout:**
- Full viewport height: `flex flex-col h-dvh` (use `dvh` for mobile viewport correctness, fallback `h-screen`)
- Header (fixed): Bot identity — "Flo" name with Bot icon, subtitle "FlowBoard AI Support", "AI assistant — answers based on documentation" disclaimer in small muted text. Sticky top with border-bottom. Compact on mobile (`py-3 px-4`), slightly larger on desktop.
- Messages area (flexible, scrollable): `flex-1 overflow-y-auto`. Contains MessageBubble list + TypingIndicator when streaming. Empty state when no messages: welcome message from Flo ("Hi! I'm Flo, FlowBoard's AI assistant. Ask me anything about FlowBoard — pricing, features, integrations, and more!") rendered as a centered card.
- Input area (fixed at bottom): ChatInput component with border-top. `p-4` padding. Max width container `max-w-3xl mx-auto` for the entire chat area so messages don't stretch too wide on large screens.

**Auto-scroll with IntersectionObserver:**
- Place a sentinel `<div ref={bottomRef} />` at the bottom of messages
- Track `isAtBottom` state via IntersectionObserver (threshold 0.1)
- Auto-scroll to bottom only when `isAtBottom` is true AND messages change
- Use `scrollIntoView({ behavior: 'smooth' })` for smooth scrolling

**Error handling:**
- When `status === 'error'`, show error banner below last message: red background, error text, "Retry" button calling `reload()`
- Error text: "Something went wrong. Please try again."

**Status-based UI:**
- `status === 'streaming'` or `status === 'submitted'`: Show TypingIndicator below messages, disable ChatInput
- `status === 'ready'`: Normal state, ChatInput enabled
- `status === 'error'`: Show error banner, ChatInput enabled (so user can try different message)

**Message limit warning:**
- Track message count from `messages.length`
- When messages.length >= 30 (15 exchanges), show a subtle banner: "Approaching conversation limit. [Start new chat]" — the link reloads the page (resets state)

**Responsive design:**
- Messages container: `max-w-3xl mx-auto w-full px-4`
- On mobile (`< md`): full-width messages, compact padding
- On desktop (`md+`): centered with max-width, more breathing room

**ChatWindow.tsx** sends message via:
