Create four client components that work together via shared config state:

**ConfigPanel.tsx** (~50 lines):
- Props: `config: WidgetConfig` and `onChange: (config: WidgetConfig) => void`
- WidgetConfig type: `{ position: "bottom-right" | "bottom-left"; theme: "light" | "dark"; greeting?: string }`
- Three controls stacked vertically with labels:
  1. Position: native `<select>` with "Bottom Right" and "Bottom Left" options
  2. Default Theme: native `<select>` with "Light" and "Dark" options
  3. Custom Greeting: `<textarea>` (2 rows, resize-none) with placeholder "Hi! How can I help you today?"
- Each control updates parent config via `onChange({ ...config, [field]: value })`
- Use dark-mode-aware Tailwind classes: `bg-white dark:bg-gray-800 border-gray-300 dark:border-gray-700` etc.
- Heading: "Widget Configuration" in `text-sm font-medium`

**CodeBlock.tsx** (~100 lines):
- Props: `code: string`
- Contains inline `tokenizeHTML()` function that parses HTML into tokens of type: tag, attr, string, text
- Regex pattern: `/<(\w+)|<\/(\w+)>|(\w[\w-]*)=|"([^"]*)"|([^<>"=]+)/g`
- Token colors (FIXED, not theme-dependent — code block is always dark):
  - tag: `text-pink-400` (e.g., `<script`, `</script>`, `>`)
  - attr: `text-cyan-300` (e.g., `src=`, `data-theme=`)
  - string: `text-green-300` (e.g., `"https://..."`)
  - text/whitespace: `text-gray-400`
- Code block wrapper: `bg-gray-900 rounded-lg p-4` with `relative` positioning (ALWAYS dark background regardless of admin theme)
- Copy button positioned `absolute top-3 right-3`:
  - Uses `navigator.clipboard.writeText()` with error handling for non-HTTPS
  - Shows `Copy` icon (lucide) + "Copy" text normally
  - On click: swaps to `Check` icon (lucide, `text-green-400`) + "Copied!" for 2000ms via setTimeout
  - Button style: `px-3 py-1.5 rounded-lg bg-gray-800 hover:bg-gray-700 text-gray-300 transition-colors text-sm`
- Render tokens via `.map()` with `<span>` elements

**BrowserPreview.tsx** (~60 lines):
- Props: `config: WidgetConfig`
- Browser mockup structure:
  1. Outer wrapper: `border rounded-lg overflow-hidden shadow-xl` with dark mode support
  2. Toolbar: `bg-gray-100 dark:bg-gray-800 px-4 py-2.5 border-b` containing:
     - Three macOS window dots (red, yellow, green circles, `w-3 h-3 rounded-full`)
     - Fake address bar: `bg-white dark:bg-gray-900 rounded-md px-4 py-1.5 text-sm` showing "https://example.com"
  3. Content area: `h-[500px]` (not 600px — leaves room for toolbar) containing:
     - `<iframe>` pointing to `/widget?theme={config.theme}&position={config.position}` if we use URL params
     - OR just `/widget` with postMessage config updates (Plan 2 handles this)
     - For now: src=`/widget` with `className="w-full h-full border-0"`, title="Widget Preview"
     - `allow="clipboard-write"`
- Use `useRef` for iframe reference (needed for Plan 2 postMessage)
- Use `useEffect` watching `config` to send postMessage to iframe: `{ type: "CONFIG_UPDATE", config }`
  - Include load event listener on iframe to send config after iframe loads
  - Target origin: `window.location.origin`
