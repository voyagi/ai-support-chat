  <name>Task 1: Create streaming chat API route with RAG, citations, and persistence</name>

  <action>
Create `src/app/api/chat/route.ts` with a single exported `POST` handler.

**Request shape:**
```typescript
{
  messages: Array<{ role: 'user' | 'assistant'; content: string }>;
  conversationId: string | null;
}
```

**Flow (in order):**

1. **Parse request body** -- extract `messages` and `conversationId` from JSON.

2. **Validate input:**
   - `messages` must be non-empty array
   - Last message must have role `'user'`
   - Return 400 JSON error if invalid

3. **Check message cap:**
   - If `conversationId` is provided, call `getMessageCount(conversationId)`
   - If count >= `MAX_MESSAGES_PER_CONVERSATION`, return 400 JSON:
     `{ error: "Conversation limit reached. Please start a new conversation." }`

4. **Create conversation if needed:**
   - If `conversationId` is null, call `createConversation()` to get a new UUID
   - This MUST happen BEFORE streaming (per research: race condition prevention)
   - Store the final `conversationId` for response metadata

5. **RAG retrieval:**
   - Extract last user message content
   - Call `searchSimilarChunks(userMessage, { threshold: 0.7, count: 5 })`
   - Select top 1-2 chunks as `citationSources` (for source card metadata)
   - Format all retrieved chunks with `formatRagContext(chunks)`

6. **Build context:**
   - Build system prompt: `buildSystemPrompt(ragContext)`
   - Count tokens for system prompt and RAG context using `countTokens`
   - Calculate available budget: `calculateAvailableBudget(systemTokens, ragTokens)`
   - Select history messages: `selectHistoryMessages(messages, availableBudget)`

7. **Stream response:**
   - Use Vercel AI SDK `createOpenAI` from `'ai'` package (NOT the existing `openai` singleton) to create a provider instance:
     ```typescript
     import { createOpenAI } from '@ai-sdk/openai'
     const openaiProvider = createOpenAI({ apiKey: process.env.OPENAI_API_KEY })
     ```
   - Note: the AI SDK has its own OpenAI provider (`@ai-sdk/openai`), which is included as a dependency of `ai`. Use this instead of the raw `openai` package for `streamText`.
   - Call `streamText()` with:
     - `model: openaiProvider('gpt-4.1-mini')` -- per research, GPT-4o-mini retiring Feb 13
     - `system: systemPrompt`
     - `messages: selectedHistory` -- the token-budgeted history
     - `abortSignal: req.signal` -- enables stop button (locked decision)
     - `maxTokens: 1024` -- reasonable per-response limit (Claude's discretion)
     - `temperature: 0.7` -- balanced creativity (Claude's discretion)
     - `onFinish: async ({ text }) => { ... }` -- fire-and-forget persistence

8. **onFinish callback (fire-and-forget):**
   - Call `saveMessages(conversationId, userMessage, text)` -- save both user msg and assistant response
   - Wrap in `.catch(err => console.error('Failed to persist conversation:', err))`
   - Do NOT await -- this is fire-and-forget per research recommendation

9. **Return streaming response with citation metadata:**
   - Use `result.toDataStreamResponse()` (this is the correct method for Vercel AI SDK v6+)
   - Before returning, write custom data to the stream using `result.mergeIntoDataStream()` or use response headers to pass citation metadata
   - Alternative approach: Set the `conversationId` and `sources` as custom response headers:
     - `X-Conversation-Id: {conversationId}`
   - For source citations, use the AI SDK's data stream protocol to send metadata. The recommended approach for v6 is to use `streamText` with `onFinish` and send sources as a separate mechanism.
   - **Simplest approach for citations:** Return sources in a custom header as JSON:
     - `X-Sources: {JSON.stringify(citationSources.map(c => ({ title: c.documentTitle, heading: c.sectionHeading, snippet: c.content.substring(0, 200), similarity: c.similarity })))}`
   - Return the stream response with these headers added

**Error handling:**
- Wrap the entire handler in try/catch
- Pre-stream errors (validation, RAG search failure): Return 500 JSON `{ error: 'Failed to generate response. Please try again.' }`
- Mid-stream errors are handled by Vercel AI SDK automatically (sends error SSE event)
- Log all errors with `console.error` including context about what failed

**Important implementation notes:**
- Import `streamText` from `'ai'` and `createOpenAI` from `'@ai-sdk/openai'`
- The `@ai-sdk/openai` package is automatically installed as a dependency of `ai`
- Do NOT use the existing `openai` client from `src/lib/openai.ts` for streaming -- the AI SDK needs its own provider
- Use `toDataStreamResponse()` (NOT `toUIMessageStreamResponse()` which may not exist in all versions)
