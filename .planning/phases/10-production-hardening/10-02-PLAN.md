  <action>
1. Create `supabase/migrations/010_sandbox_tenant_isolation.sql`:
   - Add `tenant_id TEXT` column to `documents` table (nullable, NULL = main demo KB)
   - Add `tenant_id TEXT` column to `document_chunks` table (nullable)
   - Create indexes on both tables for tenant_id filtering:
     ```sql
     CREATE INDEX IF NOT EXISTS documents_tenant_id_idx ON documents(tenant_id);
     CREATE INDEX IF NOT EXISTS document_chunks_tenant_id_idx ON document_chunks(tenant_id);
     ```
   - Update the `match_document_chunks` RPC function to accept an optional `tenant_id` parameter. When provided, return chunks where tenant_id IS NULL (main KB) OR tenant_id matches the parameter. When NULL, return only main KB chunks (tenant_id IS NULL). This is SIMPLER than RLS and avoids the complexity of set_config in serverless.
   - NOTE: Do NOT use RLS with set_config. The research suggested it, but serverless environments make set_config unreliable. Instead, filter by tenant_id directly in queries.

2. Create `src/lib/sandbox/tenant-id.ts`:
   - Import `createHash` from "crypto"
   - Export `getTenantIdFromIp(ip: string): string`:
     - Hash IP with sha256
     - Return `tenant_${hash.substring(0, 12)}`
   - Export `getIpFromRequest(request: Request): string`:
     - Try `request.headers.get("x-forwarded-for")` first (split on comma, take first)
     - Fallback to "127.0.0.1"

3. Create `src/lib/sandbox/upload-limits.ts`:
   - Export `SANDBOX_LIMITS` constant:
     ```
     { maxDocuments: 3, maxPagesPerDoc: 10, allowedTypes: ['.txt', '.md', '.pdf'], maxFileSizeMb: 5 }
     ```
   - Export `checkSandboxLimits(tenantId: string): Promise<{ allowed: boolean, currentCount: number, maxDocuments: number, error?: string }>`:
     - Query documents table for count where tenant_id = tenantId
     - Return allowed: currentCount < maxDocuments

4. Create `src/app/api/sandbox/upload/route.ts`:
   - Export POST handler
   - Check `process.env.NEXT_PUBLIC_SANDBOX_ENABLED !== 'true'` -> return 403 "Sandbox mode is disabled"
   - Extract IP from request headers (getIpFromRequest)
   - Generate tenant ID (getTenantIdFromIp)
   - Check upload limits (checkSandboxLimits)
   - If limit reached: return 429 "Document limit reached (3 max)"
   - Parse multipart form data (file from FormData)
   - Validate file type (.txt, .md only for v1, PDF adds complexity)
   - Validate file size (max 5MB)
   - Insert document into `documents` table WITH tenant_id set
   - Chunk the content using existing `chunkMarkdown`
   - Generate embeddings using existing `generateEmbeddings`
   - Insert chunks into `document_chunks` WITH tenant_id set
   - Orphan cleanup if embedding fails (same pattern as admin upload)
   - Return { success: true, documentId, chunkCount }
  </action>

    - Upload route checks NEXT_PUBLIC_SANDBOX_ENABLED before allowing uploads

  </verify>

    - Sandbox upload API validates limits, sets tenant_id, chunks, embeds
