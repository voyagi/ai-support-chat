2. Create `src/lib/rate-limit.ts`:
   - Import `Ratelimit` from `@upstash/ratelimit`, `Redis` from `@upstash/redis`
   - Create and export `ratelimit` singleton OUTSIDE handler for edge caching:
     ```
     Ratelimit({
       redis: Redis.fromEnv(),
       limiter: Ratelimit.slidingWindow(20, "1 h"),
       analytics: true,
       prefix: "@upwork-ai-chatbot",
     })
     ```
   - Export `checkRateLimit(identifier: string)` function that calls `ratelimit.limit(identifier)` and returns `{ success, limit, remaining, reset }`
   - Export `getIpAddress(request: NextRequest): string` utility:
     - Try `request.ip` first (Vercel provides this)
     - Fallback to `x-forwarded-for` header (first IP in comma-separated list)
     - Last resort: "127.0.0.1"

3. Create `src/lib/cost-tracking.ts`:
   - Import `Redis` from `@upstash/redis`
   - Create Redis client via `Redis.fromEnv()`
   - Define PRICING object for gpt-4.1-mini:
     - input: $0.15 / 1,000,000 tokens
     - output: $0.60 / 1,000,000 tokens
   - Define DAILY_BUDGET = 10 (dollars)
   - Define ALERT_THRESHOLDS: warning (50%, $5), critical (80%, $8), shutoff (100%, $10)
   - Export `getCurrentCost(): Promise<number>`:
     - Get today's date string (YYYY-MM-DD)
     - Read `cost:{date}` key from Redis
     - Return 0 if not found or different day
   - Export `trackChatCost(inputTokens: number, outputTokens: number): Promise<void>`:
     - Calculate cost from tokens * pricing
     - Atomically increment Redis `cost:{date}` key (use `redis.incrbyfloat`)
     - Set TTL to 172800 (2 days) if key is new
   - Export `checkBudgetRemaining(): Promise<{ allowed: boolean, current: number, budget: number, percentUsed: number }>`:
     - Get current cost, calculate percent, return allowed status
   - Export `checkCostAlerts(currentCost: number): { level: 'ok' | 'warning' | 'critical' | 'shutoff', message: string }`:
     - Compare against thresholds, return appropriate level and message
  </action>

<task type="auto">
  <name>Task 2: Add rate limiting to middleware and cost tracking to chat API</name>
  <files>

  <action>
1. Update `src/middleware.ts`:
   - Keep existing admin auth redirect logic (lines 15-26)
   - Add rate limiting for `/api/chat` requests:
     - Import `checkRateLimit`, `getIpAddress` from `@/lib/rate-limit`
     - For requests matching `/api/chat`:
       - Extract IP via getIpAddress(request)
       - Call checkRateLimit(ip)
       - If not success: return `NextResponse.json({ error: "Rate limit exceeded", remaining: 0, reset: new Date(reset).toISOString(), retryAfterSeconds: Math.ceil((reset - Date.now()) / 1000) }, { status: 429 })`
       - If success: add headers to response:
         - `X-RateLimit-Limit`: limit.toString()
         - `X-RateLimit-Remaining`: remaining.toString()
         - `X-RateLimit-Reset`: new Date(reset).toISOString()
   - Update matcher to include both admin and API routes:
     ```
     matcher: ["/admin/:path*", "/api/chat"]
     ```

   IMPORTANT: Rate limiting should only fire on `/api/chat` POST requests (the actual message sends), NOT on page loads like `/chat` or widget page loads. This prevents burning rate limit on page navigations.

2. Update `src/app/api/chat/route.ts`:
   - Import `checkBudgetRemaining`, `trackChatCost` from `@/lib/cost-tracking`
   - Import `countTokens` from `@/lib/embeddings/token-counter` (already imported)
   - Add budget check at the VERY TOP of the POST handler (before any other logic):
     ```
     const budgetCheck = await checkBudgetRemaining();
     if (!budgetCheck.allowed) {
       return Response.json({
         error: "Demo temporarily unavailable due to high usage. Please try again tomorrow.",
         budgetExceeded: true,
       }, { status: 503 });
     }
     ```
   - In the `onFinish` callback of `streamText`, add cost tracking AFTER persistence:
     ```
     const outputTokens = countTokens(text);
     trackChatCost(inputTokens, outputTokens).catch(err => {
       console.error("Failed to track cost:", err);
     });
     ```
     Where `inputTokens` is calculated from system prompt + user messages (use the existing `systemTokens` + `ragTokens` + count of selected history messages).
   - Cap `maxOutputTokens` at 300 (was 1024) per user decision "Responses are capped at 300 tokens to control OpenAI costs"
  </action>
