1. Create `src/lib/rag/evaluation.ts`:

Export functions for RAG evaluation metrics:

```typescript
export function calculatePrecisionAtK(
  retrievedIds: string[],
  relevantIds: string[],
  k: number
): number
```
- Take the top K retrieved chunk IDs
- Count how many are in the relevant set
- Return count / k (0.0 to 1.0)
- Handle edge case: if relevant set is empty, return 0

```typescript
export function calculateRecallAtK(
  retrievedIds: string[],
  relevantIds: string[],
  k: number
): number
```
- Take the top K retrieved chunk IDs
- Count how many of the relevant set were found
- Return count / relevantIds.length (0.0 to 1.0)
- Handle edge case: if relevant set is empty, return 1.0 (nothing to find = found everything)

```typescript
export interface SimilarityScoreReport {
  totalResults: number;
  resultsAboveThreshold: number;       // count of results with similarity >= 0.7
  percentAboveThreshold: number;       // 0.0 to 1.0
  queriesWithWeakTopResult: string[];  // query texts where top result similarity < 0.7
  avgTopResultSimilarity: number;      // average similarity of the #1 result across all queries
  minTopResultSimilarity: number;      // lowest top-result similarity across all queries
}

export interface EvaluationResult {
  queryId: string;
  query: string;
  category: string;
  difficulty: string;
  precisionAt5: number;
  recallAt20: number;
  topResultSimilarity: number;         // similarity score of the #1 result
  resultsAboveThreshold: number;       // how many of top-5 results have similarity >= 0.7
  retrievedDocuments: string[];        // document titles of top 5 results
  expectedDocuments: string[];         // from ground truth
  passed: boolean;                     // this individual query passed
}

export interface EvaluationSummary {
  totalQueries: number;
  avgPrecisionAt5: number;
  avgRecallAt20: number;
  passed: boolean;  // overall gate: avgPrecision >= 0.7 AND avgRecall >= 0.8
  similarityReport: SimilarityScoreReport;  // similarity score distribution
  byCategory: Record<string, { avgPrecision: number; avgRecall: number; count: number }>;
  byDifficulty: Record<string, { avgPrecision: number; avgRecall: number; count: number }>;
  results: EvaluationResult[];
}
```

```typescript
export async function evaluateRetrieval(
  testCases: TestCase[],
  searchFn: (query: string, count: number) => Promise<{ documentTitle: string; id: string; similarity: number }[]>
): Promise<EvaluationSummary>
```

Implementation:
- For each test case, call searchFn with count=20 (need top 20 for recall)
- Determine relevance: a retrieved chunk is "relevant" if its `documentTitle` matches one of the `expectedDocuments` (strip the `.md` extension and convert filename to title, e.g., `flowboard-pricing.md` -> match any chunk whose documentTitle was extracted from that file). The simplest approach: the document title in the database matches the H1 heading from the fixture. So compare retrieved `documentTitle` against expected titles.

Actually, since the evaluation queries reference filenames (e.g., "flowboard-pricing.md") but the database stores document titles (e.g., "FlowBoard Pricing"), the evaluate script needs a mapping. Two approaches:
  a. The evaluate script reads the fixture files to extract their H1 titles, builds a filename->title map
  b. The evaluation queries use document titles directly

Use approach (a): read fixture files, extract H1 titles, build the map. This keeps the evaluation queries referencing stable filenames.

- Calculate precision@5 and recall@20 for each query
- A query "passes" if precision@5 >= 0.7 AND recall@20 >= 0.8
- Aggregate: average precision@5 and recall@20 across all queries
- Group by category and difficulty for breakdown reporting
- Overall passes if avgPrecision >= 0.7 AND avgRecall >= 0.8

**Similarity score tracking (IMPORTANT -- addresses success criterion #4):**
- For each query result, record `topResultSimilarity` (the similarity score of the #1 result from searchFn)
- For each query, count how many of the top-5 results have `similarity >= 0.7` and store as `resultsAboveThreshold`
- After all queries, build the `SimilarityScoreReport`:
  - `totalResults`: sum of all top-5 results across all queries
  - `resultsAboveThreshold`: count of all results with similarity >= 0.7
  - `percentAboveThreshold`: resultsAboveThreshold / totalResults
  - `queriesWithWeakTopResult`: list of query strings where the #1 result had similarity < 0.7
  - `avgTopResultSimilarity`: average of all queries' top result similarity
  - `minTopResultSimilarity`: minimum across all queries' top result similarity
- Attach this report to `EvaluationSummary.similarityReport`
