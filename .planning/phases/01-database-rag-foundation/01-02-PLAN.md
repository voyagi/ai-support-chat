Create `test/fixtures/evaluation-queries.json` with 18 test query-answer pairs for evaluating RAG retrieval quality.

JSON structure:
```json
{
  "description": "Ground truth test cases for RAG evaluation. Each query maps to expected source documents and topics.",
  "testCases": [
    {
      "id": "eval-01",
      "query": "How much does FlowBoard cost?",
      "category": "pricing",
      "expectedDocuments": ["flowboard-pricing.md"],
      "expectedTopics": ["pricing", "tiers", "cost"],
      "expectedAnswer": "FlowBoard offers three tiers: Free (up to 5 users), Pro ($12/user/month), and Enterprise ($29/user/month). Annual billing saves 20%.",
      "difficulty": "easy"
    }
  ]
}
```

Create 18 test cases across these categories:

**Easy (6 queries)** — Direct questions with clear single-document answers:
- Pricing: "How much does FlowBoard cost?"
- Features: "Does FlowBoard have Kanban boards?"
- Getting started: "How do I create my first project?"
- Security: "Is FlowBoard SOC 2 compliant?"
- Mobile: "Is there a FlowBoard mobile app?"
- API: "What is the FlowBoard API rate limit?"

**Medium (6 queries)** — Questions requiring inference or spanning related content:
- "Can I import my Jira projects into FlowBoard?" (integrations)
- "What happens when I downgrade from Pro to Free?" (billing FAQ)
- "How do I give a client view-only access?" (team management - guest access)
- "What integrations work with FlowBoard?" (integrations - overview)
- "How do I set up two-factor authentication?" (security)
- "Can I use FlowBoard offline on my phone?" (mobile - offline mode)

**Hard (6 queries)** — Edge cases: typos, conversational phrasing, compound questions, cross-document:
- "whats the cheepest plan" (typo + conversational pricing query)
- "I need something for my team of 20 with SSO, what do you recommend?" (cross-doc: pricing Enterprise + security SSO)
- "how much storage do I get and can I connect to github" (compound: features + integrations)
- "We're worried about GDPR - is our data safe in FlowBoard?" (conversational security query)
- "flowboard vs asana pricing" (competitive comparison - may not have direct answer)
- "tell me everything about the enterprise plan" (broad query, multiple documents)

For each test case:
- `expectedDocuments`: list the fixture filenames that should appear in top results
- `expectedTopics`: 2-4 topic keywords the retrieved chunks should contain
- `expectedAnswer`: a brief summary of what the correct answer should include (for human verification)
- `difficulty`: "easy", "medium", or "hard"
- `category`: topic category for reporting

NOTE: The `relevantChunkIds` field (needed for precision/recall calculation) will be populated AFTER seeding, when chunk IDs are known. The evaluation script will map `expectedDocuments` to actual chunk IDs at runtime by looking up chunks that belong to those documents.
