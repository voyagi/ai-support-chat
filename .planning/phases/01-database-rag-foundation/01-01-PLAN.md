1. Create `src/lib/embeddings/types.ts` with shared types:

```typescript
export interface Chunk {
  content: string;           // The chunk text (section heading prepended)
  documentTitle: string;     // Parent document title for citation
  sectionHeading: string;    // H2 heading this chunk belongs to
  position: number;          // Chunk N of M (1-indexed)
  totalChunks: number;       // Total chunks in document (set after all chunks created)
  tokenCount: number;        // Actual token count of content
}

export interface ChunkOptions {
  targetTokens: number;      // Target chunk size (default 500)
  overlapPercent: number;    // Overlap when subdividing (default 0.15)
  minChunkTokens: number;    // Minimum chunk size to keep (default 50)
}

export const DEFAULT_CHUNK_OPTIONS: ChunkOptions = {
  targetTokens: 500,
  overlapPercent: 0.15,
  minChunkTokens: 50,
};
```

2. Create `src/lib/embeddings/token-counter.ts`:
- Export `countTokens(text: string): number` — uses gpt-tokenizer's `encode` and returns `.length`
- Export `isWithinTokenLimit(text: string, limit: number): boolean`
- Import from `gpt-tokenizer` (it uses o200k_base encoding by default which matches GPT-4o)

3. Create `src/lib/embeddings/chunker.ts`:
- Export `chunkMarkdown(document: { title: string; content: string }, options?: Partial<ChunkOptions>): Chunk[]`

Chunking algorithm:
a. Split content at `## ` heading boundaries (level 2 headings). The text BEFORE the first heading is treated as an "Introduction" section.
b. For each section, extract the heading text (e.g., "## Pricing" -> "Pricing").
c. Detect FAQ sections: if a section contains multiple `### ` sub-headings where each sub-section is under the target token count, treat each Q&A pair as its own chunk (preserve as single chunks even if small).
d. For non-FAQ sections: if the section exceeds targetTokens, subdivide at paragraph boundaries (`\n\n`). If a paragraph itself exceeds targetTokens, subdivide at sentence boundaries (`. ` followed by uppercase letter or newline).
e. When subdividing, apply overlap: the last `overlapPercent` of the previous chunk's tokens are prepended to the next chunk. Overlap at paragraph/sentence boundary, not mid-word.
f. Skip chunks below minChunkTokens (too small to be useful).
g. Prepend section heading to chunk content: `"{sectionHeading}: {chunkContent}"`. This is the content that will be embedded.
h. After all chunks are created, set `totalChunks` on every chunk to the final count.
i. Set `position` as 1-indexed sequential across the entire document.

Important implementation details:
- Use a proper heading split: regex `/^## /m` to split, but be careful NOT to split on `##` inside code blocks (fenced with triple backticks). A simple approach: strip code blocks before splitting, then restore. OR: split line-by-line, track code block state.
- Handle edge cases: empty document, document with no headings (treat entire content as one section with heading "General"), document with only H1 (treat as no-heading document).
- The `content` field in the Chunk interface stores the heading-prepended version. This is what gets embedded.

Do NOT use LangChain or any external chunking library. This is hand-rolled per research recommendation (heading-aware logic is domain-specific).
